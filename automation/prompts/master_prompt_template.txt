## 6. Security Fix Requirements

**ALL secure versions MUST include:**

1. **Primary defense** (parameterized queries, input validation, etc.)
2. **Why it works** (explain the security mechanism)
3. **Defense in depth** (multiple layers)
4. **Error handling** (fail securely)
5. **Logging** (for monitoring)

**Example:**
```python
# PRIMARY DEFENSE: Parameterized query
sql = "SELECT * FROM users WHERE id = %s"
cursor.execute(sql, (user_id,))  # Safe: query structure is fixed

# DEFENSE IN DEPTH: Input validation
if not user_id.isdigit():
    abort(400, "Invalid user ID")

# ERROR HANDLING: Fail securely
try:
    result = cursor.fetchone()
except Exception as e:
    app.logger.error(f"Database error: {{e}}")
    abort(500, "Internal server error")  # Don't leak details
```
